Register New Bot

AI Arena bots are ladder-packaged bots that run via the aiarena/local-play-bootstrap Docker infrastructure. They can be written in any language.

Place the bot directory in test_lab/aiarena/bots/. You can download bots from aiarena.net (toggle "Only downloadable").

Build configs: To configure named builds for a bot, create subdirectories under test_lab/aiarena/configs/<bot_directory>/<build_name>/. Files in a build directory are overlaid onto the bot's root at runtime via Docker volume mounts. See test_lab/aiarena/configs/README.md for details.

To add a bot:

  • add the ladder-packaged version to the bots directory.
  • Refresh this page, your bot should appear in the list below of available bot directories. Click the name to autopopulate the form.
To use your live development code instead of the ladder-packaged version:
  • Check the box to flag it as a "test bot".
  • Change the source path to point to your code instead of the ladder-packaged version.

Replay Tests

Select a replay file and define test parameters. You can add multiple tests from the same replay before submitting.

Replay tests set two environment variables that bots can read at runtime:
REPLAY_TAKEOVER_TIME — the game time (in seconds, float) at which the bot took over from the replay. Bots should use this to offset self.time so game-time-dependent logic (e.g. build order timing) works correctly. Can be used as a flag for adjusting other logic to work with Replay Tests
REPLAY_DURATION — how many seconds after takeover the test should run before the bot forfeits. This allows testing small chunks of gameplay

Paths

Configure local paths for your StarCraft II installation and match output. These can also be set during first-run setup.

Required.Host path to your StarCraft II Maps directory. Example: C:\Program Files (x86)\StarCraft II\Maps Not required to have a local install of SC2 to use framework, but should use that Maps dir if it exists, for replay watching.
Path to SC2Switcher.exe for watching replays. Inside the Support folder (next to Maps) of SC2 local install

Concurrency

Control how many custom bot instances can run simultaneously. Each match uses one or two slots: vs Blizzard AI and replay tests count as 1; vs custom bot and vs past version count as 2. When the limit is reached, new matches are queued and started automatically as running matches finish.

0 = unlimited (no limit on parallel matches)
